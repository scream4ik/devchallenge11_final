General information

The development of the project used the MTV pattern (Model-Template-View). This design pattern is very close to the classic MVC, and most importantly, what it allows is a good way to separate business logic from design.

Start project

This project is started with the Docker software. The project is configured in debug mode, its current settings are suitable for local development, as well as for server development. For the production server, you need to slightly expand the project settings.

To create an image and lift the containers, execute the command:

docker-compose up -d --build
(Alternate command: make up)

Now the project is available at 127.0.0.1:8000

API

Registration new employee example:

http://127.0.0.1:8000/register?name=user1&area=bills&area=contracts

Example of list area and employees:

http://127.0.0.1:8000/call?area=moto&area=bills&area=leases&area=contracts&area=bills

Employees status reset example:

http://127.0.0.1:8000/reset

Tests

There are unit tests in the project that cover the API. To run the tests, run the command:

docker-compose exec web python3 project/manage.py test project/
(Alternate command: make test)

Panel administration

You can found panel administration located at 127.0.0.1:8000/admin/ with next credentials: username - admin, password - adminadmin.
